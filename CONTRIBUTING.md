# Contributing

## Propose changes
- Open an issue or discussion for non-trivial changes.
- Create a feature branch from the default branch.
- Submit a PR with a clear summary and scope.

## Code style expectations
- Keep changes focused; avoid massive refactors in one PR.
- Follow existing conventions and formatting.
- Prefer small, reviewable increments.

## Commits and PR descriptions
- Write descriptive commit messages (imperative, present tense).
- Reference relevant issues or context.
- PRs should include: purpose, scope, and testing notes (if any).

## Document decisions
- Record significant decisions in docs/adr.
- Keep ADRs short and use consistent numbering.
